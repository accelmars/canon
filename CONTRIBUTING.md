# Contributing to canon

Thank you for your interest in contributing.

## Prerequisites

- Rust stable (1.70+): `rustup update stable`
- Cargo: included with Rust

## Setup

```bash
git clone https://github.com/accelmars/canon.git
cd canon
cargo build
cargo test
```

## Contribution types accepted

- Bug fixes
- Documentation improvements
- Issue reports with reproduction steps

Feature contributions: please open an issue first to discuss scope before implementing.

## Commit format

All commits must follow [Conventional Commits](https://www.conventionalcommits.org/):

```
type(scope): description
```

Types: `feat`, `fix`, `docs`, `chore`, `refactor`, `test`

Commit titles are written for an external reader in release notes. Do not include internal workflow language, contract IDs, or guild-internal references in commit titles.

## Branching

| Prefix | Use for |
|--------|---------|
| `feat/` | New features |
| `fix/` | Bug fixes |
| `docs/` | Documentation |
| `chore/` | Tooling, CI, dependencies |
| `refactor/` | Code refactoring |

## PR process

1. Branch from `main` using the prefix convention above
2. Open a PR — fill in the PR template
3. CI must pass (fmt, clippy, test, build)
4. Squash merge only — maintainer squashes on merge

## What NOT to commit

- Secrets, API keys, tokens of any kind
- Hardcoded absolute paths (`/Users/...`)
- Private repository names or internal methodology strings
- Files generated by `cargo build` (covered by `.gitignore`)
